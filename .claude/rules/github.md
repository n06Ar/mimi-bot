# GitHub 操作ルール

## 書き込み操作

GitHub への書き込み操作（PR レビュー・コメント・Issue 作成など）は必ず `~/.claude/shells/gh-as-mimi.sh` 経由で実行すること。

- 例: `~/.claude/shells/gh-as-mimi.sh pr review 10 --repo n06Ar/repo --approve --body "..."`
- 通常の `gh` コマンドは読み取り操作にのみ使用する

## webhook 通知への自動対応（auto-review）

PR 通知チャンネルに GitHub Bot から通知が届いたとき、**ユーザーへの確認なしに自動でレビューを実行する**。

### 自動レビューの流れ

1. 通知メッセージから `repo`・`PR番号` を抽出する
2. PR の diff を取得してレビューする
3. **問題なし** → スレッドを Resolve ＋ PR を Approve する
4. **問題あり** → GitHub のインラインコメント＋レビューコメントで指摘する

### 「問題あり」の基準

以下が存在する場合は「問題あり」とする：
- バグ・型エラー・ロジック的な誤り
- セキュリティリスク
- 実行時エラーになり得るコード
- 命名が動作・意図を正確に表していない

## PR レビューのスタイル

- 指摘はなるべく GitHub REST API でインラインコメントとして投稿する
- 全体サマリーは `gh-as-mimi.sh pr review ... --comment --body` で投稿する
- レビューコメントはミミのキャラクターで書く（鳴き声 + （）補足スタイル）
